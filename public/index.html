Why?

The original intention behind releases.io was a combination of the following aims:

  • To promote and encourage the use of Github Releases for open-source projects. Hosting your release notes at the same location that you host your code just makes sense. Writing release notes in Markdown is also a delightful experience.
  • To supplant the need to maintain many static HTML files for inclusion in software update frameworks such as Sparkle — when no pizzaz is required — and instead use release notes generated directly from those stored on Github.
  • To provide a quick and easy way of sharing release notes with other people, without the full-blown Github webpage interface, putting the focus squarely on the content.
  • As a quick way of accessing release notes for inclusion in other projects, in either HTML or raw Markdown.
  • For fame and profit — which may or may not be true.
